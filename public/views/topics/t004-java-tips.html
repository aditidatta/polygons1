Java Tips and Tricks

1) Alternative of Arrays.asList(T... a) for primitive type arrays

You may think it creates a List and copies all the elements from the array to the list. But that's not the case. It just creates a wrapper around the array, no iteration or copies are performed. But it doesn't work with primitive type arrays. If an int[] is sent it will just create a List object, i.e. the whole array as an element in the list. So what can we do to acheive the same with primitive type arrays? Here's what we can do: 

    int[] a = new int[]{1,2,3,4,5,6,7,8}; 
	
    List wlist = new AbstractList() {
        public Integer get(int i) {
            return a[i];
        }
        
        public int size() {
            return a.length;
        }
        
        public Integer set(int i, Integer num) {
            int val = a[i];
            a[i] = num;
            return val;
        }
    };
    // now we have a List wrapper to the int[] array

Now, there's another way to do it, that is by creating stream (we will discuss about streams soon) of the array: 

    List list = IntStream.of(a).boxed().collect(Collectors.toList());
But there's a difference between these two methods. The 1st one just creates a wrapper, so there no copy. But in the 2nd method theres a copy operation going on underneath. You can test it by modifying a[]. You will see that list object from the 2nd method will stay unchanged. That's because IntStream.boxed() actually wraps the primitive data into their object classes (like int to Integer) and that actually creates new copies of the data. Hence, the time complexity of the 2nd method is O(N), whereas the time complexity of the 1st method is O(1). But note that it needs different implementations for different types.
